7. iptables : tool for ipv4 packet filtering and NAT
    iptables -L : lists the current iptable values
    service iptables stop : to stop the firewalls in red hat
    service iptables start : starting the firewalls in red hat
    rcSuSEfirewall2 status : displays the status of the firewall in SuSE
    rcSuSEfirewall2 start : starts the firewall in SuSE
    rcSuSEfirewall2 stop : stops the firewall in SuSE
   /etc/sysconfig/iptables : contains the list of default iptables
   system-config-firewall : configuring firewall in gui
   Built-in chains for filter table:-
   INPUT : built-in chain for the packets that are targeted for the host
   FORWARD : built-in chain for the forwarded packets
   OUTPUT : built-in chain for the packets that are generated locally 
   Built-in chains for nat table:-
   PREROUTING : alters network packets when they arrive
   OUTPUT : alters locally generated packets before they are sent out
   POSTROUTING : alters network packets before they are sent out
   Built-in chains for mangle table:-
   INPUT : alters network packets that are targeted for the host
   OUTPUT : alters locally-generated network packets before they are sent out 
   FORWARD : alters network packets routed through the host
   PREROUTING : alters incoming network packets before they are routed
   POSTROUTING : alters network packets before they are sent out
   RH-Firewall-1-INPUT : user-defined chains created by security level configuration tool
       -A INPUT -j RH-Firewall-1-INPUT : any input jumps to the firewall chain
       -A FORWARD -j RH-Firewall-1-INPUT : any forwarded data jumps to this firewall chain
   ACCEPT : let the packet through
   REJECT : packet is dropped and an error message is sent to packet originator
   DROP : drops the packet and nothing is sent back to the packet originator
   QUEUE : to pass the packet to userspace
   RETURN : stop traversing this chain and resume at the next rule in the previous chain
   Parameters:-
        -j : specifies the target of the rule, what to do when the condition is matched. target can be user-defined chain like ACCEPT, REJECT or any user-defined chain
        -i : name of the interface through which packet was received
        -o : name of the interface through which packet is going to be sent 
        -s : originating address can be a ip address, hostname, network name 
        -d : destination address can be a ip address, hostname, network name 
        -p : protocol of the rule or of the packet to check. can be tcp, udp, udplite, icmp, esp, ah, sctp or all. protocol numbers as in /etc/protocols
        -m : matches the condition   
             --sport : source port
             --dport : destination port
             --state : state of the packet like NEW, INVALID, ESTABLISHED, RELATED.
        -t : for table filter, nat, mangle
     -A : append one or more rules to the end of the selected chain
     -D : delete one or more rules from the selected chain
     -I : insert one or more rules in the selected chain 
     -R : replace the rule in the selected chain
     ! : negative of the current action
